Tool Plan
Architecture:

Persistent TLE history stored in browser memory (with import/export to JSON)
SGP4 propagation via the sgp4 Python library (wrapped in a Flask API called from the frontend)
Actually, since this needs to run client-side in the artifact, I'll implement SGP4 in JavaScript using the satellite.js library (the standard JS port of SGP4)

UI Panels:

TLE Input — paste new TLE sets (with auto-parsed epoch), stored chronologically
TLE History — table of all ingested TLEs with epoch, inclination, eccentricity, BSTAR drag term
Propagation Controls — select start time, end time, time step
Ground Track / Pass Predictor — enter ground station lat/lon, compute upcoming passes
Orbit Decay Chart — plot altitude vs time across all TLE epochs + propagated future
Residuals Panel — for each TLE, show predicted position (from prior TLE propagated forward) vs actual TLE position — this is the drag/SRP model validation

Libraries: satellite.js (SGP4, CDN), Chart.js (plots), Tailwind (styling)

Five tabs, each tied to a scoring area:
Orbit Decay — plots observed altitude from each TLE epoch + SGP4-propagated future. The three race altitude bands (≈500km Hello World, active race zone, ≈300km finish) are drawn as reference lines. The "Est. reentry" metric uses observed decay rate to project when the ChipSat hits ~120km, which is the core race timeline.
Propagated Track — runs SGP4 from TLE selected in the sidebar, stepping through chosen time window. Use to schedule ground station contacts before a pass occurs.
Model Residuals — this is the core validation loop. For each consecutive TLE pair, it propagates TLE #1 forward to TLE #2's epoch, then computes how far off the predicted altitude is from what Space-Track actually observed. A residual under ~1km is reasonable error for the drag/SRP model; larger systematic residuals point to attitude or atmospheric density modeling errors.
Pass Prediction — takes TinyGS ground station coordinates and computes every pass in the next 72 hours, with max elevation, AOS/LOS times, and a quality rating. Durham NC is pre-loaded as the default.
Orbital Elements — shows the full Keplerian state for every TLE ingested, plus a BSTAR trend chart. Rising BSTAR = increasing drag = sail is descending faster.
Workflow: Click "Load sample" in the sidebar to see analytics. Export JSON to persist TLE history across sessions.
